# VetKB Final — Cluster-based Knowledge Base

> Дизайн-документ финальной версии системы базы знаний для колл-центра ветеринарной клиники.

---

## 1. Обзор

### Что это

Система автоматического накопления и структурирования знаний из реальных звонков в ветклинику. Знания организованы в **кластеры** — каждый кластер описывает одну тему обращения (стерилизация кошки, кровь в моче, вакцинация щенка и т.д.).

### Ключевые принципы

1. **Кластер = тема звонка**. Один файл — одна тема. Диалог может относиться к 1+ кластерам (one-to-many).
2. **Инкрементальное обогащение**. Каждый новый звонок дополняет существующие кластеры, а не перегенерирует их.
3. **Минимум LLM-вызовов**. Два вызова на звонок: сегментация + обогащение. Классификация — алгоритмическая (embedding similarity).
4. **Git-trackable**. Все знания — YAML-файлы в репозитории. Каждое обогащение видно через `git diff`.
5. **Одно правило для всех случаев**. Создание нового кластера, обогащение существующего, пустая KB — один и тот же алгоритм.

### Чем не является

- Это **не CRM**. Конкретные животные, владельцы, история звонков — отдельная система.
- Это **не runtime-бот**. KB — источник знаний, бот — потребитель.

## 2. Pipeline обработки звонка

```
┌──────────────────────────────────────────────────────────┐
│                    ВХОДНОЙ ЗВОНОК                         │
│                   (аудиофайл .wav/.mp3)                   │
└──────────────┬───────────────────────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────────────────────┐
│  ЭТАП 1: ТРАНСКРИПЦИЯ                                    │
│  Whisper (large-v3) → текстовый транскрипт                │
│  • Разметка: [Оператор] / [Клиент]                       │
│  • Язык: русский                                          │
│  • Без LLM — чисто ASR                                   │
└──────────────┬───────────────────────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────────────────────┐
│  ЭТАП 2: СЕГМЕНТАЦИЯ                              [LLM]  │
│  Определить: одна тема в звонке или несколько?            │
│                                                           │
│  Если одна тема → 1 сегмент (весь транскрипт)            │
│  Если несколько → разбить на N сегментов                  │
│                                                           │
│  На выходе для каждого сегмента:                          │
│  • topic: краткое название темы (2-5 слов)                │
│  • text: релевантный фрагмент транскрипта                 │
│  • urgency: emergency | urgent | routine | informational  │
└──────────────┬───────────────────────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────────────────────┐
│  ЭТАП 3: КЛАССИФИКАЦИЯ                    [Embedding]     │
│  Для каждого сегмента:                                    │
│                                                           │
│  1. Вычислить embedding сегмента                          │
│     (модель: multilingual-e5-large)                       │
│                                                           │
│  2. Cosine similarity ко всем кластерам                   │
│     (по trigger_phrases + name)                           │
│                                                           │
│  3. Решение:                                              │
│     similarity >= THRESHOLD (0.6) → существующий кластер  │
│     similarity < THRESHOLD         → НОВЫЙ кластер        │
│                                                           │
│  Без LLM — чисто алгоритмический этап                    │
└──────────────┬───────────────────────────────────────────┘
               │
               ├── сегмент → существующий кластер
               │       │
               │       ▼
               │   ┌────────────────────────────────────┐
               │   │  ЭТАП 4a: ОБОГАЩЕНИЕ        [LLM]  │
               │   │                                     │
               │   │  Вход:                              │
               │   │  • Текущий файл кластера            │
               │   │  • Новый сегмент транскрипта        │
               │   │                                     │
               │   │  Задача LLM:                        │
               │   │  • Сравнить новую информацию с       │
               │   │    существующей                      │
               │   │  • Добавить новые trigger_phrases    │
               │   │  • Дополнить шаги диалога            │
               │   │  • Добавить новые исключения         │
               │   │  • Обновить FAQ                      │
               │   │  • Добавить evidence (цитату)        │
               │   │  • Обновить _global/ если нужно      │
               │   │                                     │
               │   │  Выход: обновлённый YAML-файл       │
               │   │  (сразу apply, без review)           │
               │   └────────────────────────────────────┘
               │
               └── сегмент → новый кластер
                       │
                       ▼
                   ┌────────────────────────────────────┐
                   │  ЭТАП 4b: СОЗДАНИЕ          [LLM]  │
                   │                                     │
                   │  Вход:                              │
                   │  • Сегмент транскрипта               │
                   │                                     │
                   │  Задача LLM:                        │
                   │  • Создать новый файл кластера       │
                   │  • Заполнить все блоки на основе      │
                   │    одного сегмента                    │
                   │  • Определить category                │
                   │  • Обновить _global/ если нужно      │
                   │                                     │
                   │  Выход: новый YAML-файл              │
                   └────────────────────────────────────┘
```

### Итого на один звонок:
- 1 вызов Whisper (транскрипция)
- 1 вызов LLM (сегментация)
- N вызовов LLM (обогащение/создание, где N = количество сегментов, обычно 1-3)

**Типичный звонок с одной темой: 1 Whisper + 2 LLM вызова.**

## 3. Структура кластера

Каждый кластер — YAML-файл, описывающий одну тему обращения.

### Расположение файлов

```
kb/
├── clusters/
│   ├── urological/
│   │   ├── blood_in_urine.yaml
│   │   └── frequent_urination.yaml
│   ├── preventive/
│   │   ├── cat_sterilization.yaml
│   │   └── dog_vaccination.yaml
│   ├── emergency/
│   │   └── poisoning.yaml
│   ├── admin/
│   │   └── price_inquiry.yaml
│   └── ...
├── _global/
│   ├── clinic_info.yaml
│   ├── rules.yaml
│   ├── tone.yaml
│   └── faq.yaml
└── embeddings/
    └── cluster_index.json      # Embedding-индекс для классификации
```

### Формат файла кластера

```yaml
# ─── Метаданные ───────────────────────────────────────────
id: cat_sterilization                  # slug (английский, snake_case)
name: Стерилизация кошки               # человекочитаемое название (русский)
category: preventive                    # категория для группировки
call_count: 12                          # сколько звонков обогатило этот кластер
last_updated: 2026-03-01T14:30:00Z     # когда последний раз обогащался

# ─── Триггерные фразы ────────────────────────────────────
# Реальные фразы клиентов из звонков.
# Используются для embedding-классификации новых звонков.
trigger_phrases:
  - хотим стерилизовать кошку
  - сколько стоит стерилизация
  - кошку нужно кастрировать
  - когда лучше стерилизовать
  - подготовка к стерилизации

# ─── Шаги диалога ─────────────────────────────────────────
# Как должен идти разговор: в каком порядке, что спрашивать,
# что отвечать, какие действия предпринимать.
conversation_flow:
  - step: Выяснить возраст и породу
    ask: "Сколько вашей кошечке лет и какая порода?"
    why: "Возраст влияет на риски анестезии, порода — на доп. обследования"

  - step: Уточнить состояние здоровья
    ask: "Есть ли хронические заболевания? Прививки сделаны?"
    why: "Прививки должны быть актуальны, хронические заболевания — доп. подготовка"

  - step: Объяснить подготовку
    say: "Перед операцией нужна голодная диета 8-12 часов. Воду можно давать за 4 часа до."

  - step: Назвать стоимость и что входит
    say: "Стерилизация кошки — 5500 рублей. В стоимость входит: наркоз, операция, послеоперационная попона, наблюдение в клинике 2 часа."

  - step: Предложить дату
    action: check_slots
    doctor_role: хирург
    say: "Давайте подберём удобную дату. У нас есть..."

  - step: Подтвердить запись
    say: "Записала вас на {дата} в {время}. Не забудьте про голодную диету!"

# ─── Уточняющие вопросы ──────────────────────────────────
# Что важно уточнить у клиента. Каждый вопрос — с пояснением зачем.
clarifying_questions:
  - question: "Кошка уже рожала?"
    why: "У рожавших кошек операция сложнее, стоимость может отличаться"
    impact: "Если рожала — предупредить о возможной более высокой цене"

  - question: "Кошка сейчас не в течке?"
    why: "Во время течки операцию лучше отложить на 2 недели"
    impact: "Если в течке — перенести запись"

  - question: "Есть ли непереносимость наркоза в анамнезе?"
    why: "Редкий случай, но критически важный"
    impact: "Если да — дополнительная консультация анестезиолога"

# ─── Исключения и особые случаи ──────────────────────────
# Ситуации, отклоняющиеся от стандартного сценария.
# Могут зависеть от породы, возраста, состояния и т.д.
exceptions:
  - condition: "Порода мейн-кун"
    action: "Добавить ЭКГ и УЗИ сердца перед операцией (риск ГКМП)"
    price_impact: "+2000₽ за кардиообследование"
    source: "Рекомендация ветеринара"

  - condition: "Кошка старше 8 лет"
    action: "Обязательный биохимический анализ крови перед операцией"
    price_impact: "+1200₽ за анализ"
    source: "Протокол клиники"

  - condition: "Кошка на свободном выгуле"
    action: "Уточнить дату последней дегельминтизации, возможно нужна перед операцией"
    price_impact: "+500₽ если нужна обработка"

# ─── Услуги и цены ────────────────────────────────────────
# Какие услуги связаны с этой темой, с ценами.
services_and_prices:
  - service: "Стерилизация кошки"
    price: 5500
    currency: "₽"
    includes: "наркоз, операция, попона, наблюдение 2ч"
    mandatory: true

  - service: "ЭКГ"
    price: 1500
    currency: "₽"
    mandatory: false
    condition: "Для пород группы риска (мейн-кун, британская, шотландская)"

  - service: "УЗИ сердца"
    price: 2600
    currency: "₽"
    mandatory: false
    condition: "Для пород группы риска"

  - service: "Биохимический анализ крови"
    price: 1200
    currency: "₽"
    mandatory: false
    condition: "Кошки старше 8 лет"

# ─── Красные флаги ────────────────────────────────────────
# Когда нужно немедленно эскалировать к ветеринару / старшему администратору.
red_flags:
  - signal: "Кошка беременна"
    action: "Не записывать на стандартную стерилизацию. Перевести на ветеринара для обсуждения."
    urgency: urgent

  - signal: "Кошка после операции плохо себя чувствует (звонок post-op)"
    action: "Немедленно соединить с дежурным ветеринаром."
    urgency: emergency

# ─── Чего не говорить ─────────────────────────────────────
# Фразы и темы, которых оператор должен избегать.
never_say:
  - "Не обещать конкретный исход операции"
  - "Не говорить 'это безопасная операция' — любая операция имеет риски"
  - "Не давать медицинских рекомендаций по подготовке сверх стандартных"
  - "Не сравнивать цены с другими клиниками"

# ─── FAQ ──────────────────────────────────────────────────
# Частые вопросы клиентов в рамках этой темы.
faq:
  - q: "В каком возрасте лучше стерилизовать?"
    a: "Оптимально с 7-8 месяцев. Но можно и позже — ветеринар оценит на приёме."

  - q: "Кошка будет толстеть после стерилизации?"
    a: "Гормональный фон меняется, но при правильном питании вес контролируется. Врач даст рекомендации."

  - q: "Как быстро она восстановится?"
    a: "Обычно 7-10 дней. Швы снимают через 10-14 дней, если не саморассасывающиеся."

  - q: "Нужно ли оставлять на ночь?"
    a: "Обычно нет, через 2 часа после операции можно забирать. Но если хотите — можем оставить на стационар."

# ─── Evidence ─────────────────────────────────────────────
# Цитаты из реальных звонков. Трейсабилити знаний.
evidence:
  - call_id: "2026-01-24_14-05-18_9602355445_411331575"
    quote: "Стерилизация кошки у нас стоит 5500 рублей, туда входит наркоз, сама операция, попонка послеоперационная"
    timestamp_sec: 145.2

  - call_id: "2026-02-05_11-19-32_9039624835_424195916"
    quote: "Если мейн-кун, то мы рекомендуем перед операцией сделать ЭКГ, потому что у них бывает кардиомиопатия"
    timestamp_sec: 203.7

  - call_id: "2026-02-16_14-40-40_9777152573_436127698"
    quote: "Голодная диета минимум 8 часов перед операцией, водичку можно за 4 часа убрать"
    timestamp_sec: 89.1
```

## 4. Cross-cutting знания (_global/)

Знания, не привязанные к конкретному кластеру. LLM при обогащении сам решает, что относится к конкретному кластеру, а что к `_global/`.

### `_global/clinic_info.yaml`

```yaml
name: "ВетПомощь 24"
address: "ул. Примерная, д. 10"
phone: "+7 (495) 123-45-67"
hours:
  weekdays: "08:00 — 22:00"
  weekends: "09:00 — 20:00"
  emergency: "круглосуточно"
night_tariff:
  start: "23:00"
  end: "08:00"
  surcharge: "+500₽ к приёму"
```

### `_global/rules.yaml`

```yaml
# Правила, которые применяются ко ВСЕМ темам.
never_do:
  - "Никогда не ставить диагнозы по телефону"
  - "Никогда не назначать лечение"
  - "Никогда не отменять назначения ветеринара"
  - "Не говорить, что клиника закрыта (работаем круглосуточно для экстренных)"

always_do:
  - "Представиться: 'Клиника ВетПомощь, администратор [Имя]'"
  - "Уточнить вид животного и возраст"
  - "Предложить запись, если проблема требует осмотра"
  - "В конце — повторить дату/время записи"
```

### `_global/tone.yaml`

```yaml
# Тон общения, собранный из реальных звонков.
greeting:
  - "Добрый день! Клиника ВетПомощь, меня зовут {имя}."
  - "Здравствуйте! Ветклиника, слушаю вас."

empathy:
  - "Понимаю вашу тревогу, давайте разберёмся."
  - "Не переживайте, мы поможем."

closing:
  - "Будем ждать вас! Если что-то изменится — позвоните."
  - "Хорошего дня! Выздоравливайте."

style_notes:
  - "Говорить спокойно и уверенно"
  - "Не торопить клиента"
  - "Если клиент волнуется — сначала успокоить, потом переходить к вопросам"
```

### `_global/faq.yaml`

```yaml
# Вопросы, не привязанные к конкретной теме.
- q: "Вы работаете в выходные?"
  a: "Да, в выходные с 9 до 20, для экстренных случаев — круглосуточно."

- q: "Можно оплатить картой?"
  a: "Да, принимаем наличные и карты."

- q: "Есть ли парковка?"
  a: "Да, бесплатная парковка перед клиникой."
```

### Как LLM решает: кластер или _global?

В промте обогащения прописано правило:

> Если информация из звонка **специфична для темы** (подготовка к стерилизации, цена УЗИ) → добавить в кластер.
> Если информация **универсальна** и применима к любой теме (режим работы, способы оплаты, тон общения) → добавить в `_global/`.

## 5. Правило создания и классификации кластеров

### Единый алгоритм (работает всегда одинаково)

```
ВХОД: сегмент транскрипта (topic + text)

1. Вычислить embedding сегмента

2. Для каждого существующего кластера:
   - Вычислить similarity (cosine) между embedding сегмента
     и embedding'ами trigger_phrases + name кластера

3. best_score = max(similarities)

4. ЕСЛИ best_score >= THRESHOLD (0.6):
   → Назначить в кластер с best_score
   → ЭТАП 4a: обогащение

5. ЕСЛИ best_score < THRESHOLD:
   → ЭТАП 4b: создание нового кластера

6. ЕСЛИ несколько кластеров >= THRESHOLD (мульти-классификация):
   → Назначить во все подходящие кластеры
   → Обогатить каждый отдельно
```

### Частные случаи

| Ситуация | Что происходит |
|----------|---------------|
| **Пустая KB** | Нет кластеров → similarity = 0 → всегда создаётся новый кластер |
| **Первые 5 звонков на одну тему** | Все попадают в один кластер, обогащая его |
| **Звонок на новую тему при непустой KB** | similarity < 0.6 → новый кластер |
| **Мульти-тема: стерилизация + УЗИ** | 2 сегмента → каждый классифицируется отдельно |

### Параметры (настраиваемые)

| Параметр | Значение по умолчанию | Описание |
|----------|----------------------|----------|
| `SIMILARITY_THRESHOLD` | 0.6 | Порог для попадания в существующий кластер |
| `EMBEDDING_MODEL` | multilingual-e5-large | Модель для embedding |
| `MAX_TRIGGER_PHRASES` | 50 | Макс. триггерных фраз на кластер (для скорости) |

### Индекс кластеров

Для быстрой классификации поддерживается файл `kb/embeddings/cluster_index.json`:

```json
{
  "model": "multilingual-e5-large",
  "clusters": {
    "cat_sterilization": {
      "category": "preventive",
      "embeddings": [[0.012, -0.034, ...], ...]
    },
    "blood_in_urine": {
      "category": "urological",
      "embeddings": [[0.045, 0.023, ...], ...]
    }
  },
  "updated_at": "2026-03-01T14:30:00Z"
}
```

Перестраивается автоматически при добавлении/обновлении trigger_phrases.

## 6. Промты для LLM

### Промт сегментации (Этап 2)

```
Ты — аналитик звонков ветеринарной клиники.

Тебе дан транскрипт звонка. Определи, сколько РАЗНЫХ тем обсуждается в звонке.

Тема — это отдельная медицинская проблема или услуга:
- "стерилизация кошки" — одна тема
- "кровь в моче" — другая тема
- "стерилизация + нужно сначала сделать УЗИ для подготовки" — ДВЕ темы

НЕ считай разными темами:
- Уточняющие вопросы в рамках одной темы
- Обсуждение цены за конкретную услугу
- Организационные моменты (запись, время)

ФОРМАТ ОТВЕТА (строго JSON):
{
  "segments": [
    {
      "topic": "краткое название (2-5 слов)",
      "text": "релевантный фрагмент транскрипта (не весь!)",
      "urgency": "emergency | urgent | routine | informational"
    }
  ]
}

Если в звонке одна тема — верни один сегмент с полным транскриптом в text.
```

### Промт обогащения кластера (Этап 4a)

```
Ты — аналитик звонков ветеринарной клиники.

Тебе даны:
1. ТЕКУЩИЙ ФАЙЛ КЛАСТЕРА (YAML) — то, что мы уже знаем по этой теме
2. НОВЫЙ СЕГМЕНТ — фрагмент нового звонка на эту тему

ЗАДАЧА: Обновить файл кластера, добавив новую информацию из сегмента.

ПРАВИЛА:
- НЕ удаляй существующую информацию
- Добавляй ТОЛЬКО то, чего ещё нет в кластере
- Новые trigger_phrases — только РЕАЛЬНЫЕ фразы из транскрипта (не придумывать)
- Новые шаги диалога — только если выявлен новый паттерн
- Новые исключения — только если звонок содержит реальный пример
- Новые FAQ — только если клиент задал вопрос, которого ещё нет
- Добавь evidence с цитатой из транскрипта
- Увеличь call_count на 1
- Обнови last_updated

ЕСЛИ информация из звонка УНИВЕРСАЛЬНА (режим работы, оплата, тон):
- Не добавляй в кластер
- Вместо этого добавь в блок _global_updates

ФОРМАТ ОТВЕТА:
Верни полный обновлённый YAML-файл кластера.

В конце добавь блок:
_global_updates:
  file: "rules.yaml"  # или tone.yaml, faq.yaml, clinic_info.yaml
  additions:
    - ...
(Если обновлений для _global/ нет — не добавляй этот блок.)
```

### Промт создания нового кластера (Этап 4b)

```
Ты — аналитик звонков ветеринарной клиники.

Тебе дан СЕГМЕНТ ЗВОНКА на тему, для которой ещё нет кластера в базе знаний.

ЗАДАЧА: Создать новый файл кластера на основе этого звонка.

ПРАВИЛА:
- id: английский slug (snake_case), отражающий тему
- name: русское название темы
- category: одна из категорий (urological, preventive, emergency, admin,
  dermatology, gi, musculoskeletal, respiratory, reproductive,
  ophthalmology, dental, oncology, cardiology, neurology, general)
- trigger_phrases: ТОЛЬКО реальные фразы из транскрипта
- conversation_flow: шаги, которые видны из данного звонка
  (может быть неполным — дополнится из будущих звонков)
- Все остальные блоки заполнить тем, что есть в звонке
  (если для блока нет данных — оставить пустым списком)
- Обязательно добавить evidence

ФОРМАТ: полный YAML-файл кластера (см. схему).
```

## 9. Порядок реализации

Поэтапный план, каждый этап — самостоятельно рабочий.

### Фаза 1: Pipeline (ядро)

1. **Транскрипция** — Whisper wrapper (вход: аудио, выход: JSON с текстом)
2. **Сегментация** — LLM prompt (вход: транскрипт, выход: сегменты)
3. **Создание кластера** — LLM prompt (вход: сегмент, выход: YAML-файл)
4. **Embedding-индекс** — построение/обновление индекса из trigger_phrases
5. **Классификация** — cosine similarity к индексу (вход: сегмент, выход: кластер или "новый")
6. **Обогащение** — LLM prompt (вход: кластер YAML + сегмент, выход: обновлённый YAML)
7. **Оркестратор** — run_pipeline.py: аудио → транскрипт → сегменты → классификация → обогащение/создание

### Фаза 2: UI

1. **Главная страница** — список кластеров + детальная статья
2. **Карта кластеров** — d3/ReactFlow визуализация по категориям
3. **Граф диалога** — ReactFlow + Dagre для conversation_flow

### Фаза 3: Тюнинг

1. Подбор `SIMILARITY_THRESHOLD` на реальных данных
2. Улучшение промтов по результатам обработки
3. Валидация качества обогащения (git diff review)
